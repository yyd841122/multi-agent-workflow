#### 3. JavaScript 逻辑（script.js）

**（1）获取空状态元素**
```javascript
const emptyState = document.getElementById('emptyState');
```

**（2）创建更新空状态的函数**
```javascript
// 更新空状态显示/隐藏
function updateEmptyState() {
    // 检查任务列表中是否有任务
    if (taskList.children.length === 0) {
        // 没有任务，显示空状态提示
        emptyState.style.display = 'block';
    } else {
        // 有任务，隐藏空状态提示
        emptyState.style.display = 'none';
    }
}
```

**实现要点**：
- 使用 `taskList.children.length` 判断是否有任务
- `length === 0` 表示没有任务，显示空状态
- `length > 0` 表示有任务，隐藏空状态
- 通过修改 `style.display` 控制显示/隐藏

**（3）在添加任务后更新空状态**
```javascript
// 将列表项添加到任务列表
taskList.appendChild(li);

// 清空输入框
taskInput.value = '';

// 让输入框重新获得焦点
taskInput.focus();

// 更新空状态显示
updateEmptyState();
```

**实现要点**：
- 在 `addTask()` 函数末尾调用 `updateEmptyState()`
- 添加第一个任务后，空状态自动隐藏

**（4）在删除任务后更新空状态**
```javascript
if (event.target.className === 'delete-button') {
    // 获取要删除的列表项（删除按钮的父元素）
    const li = event.target.parentElement;
    // 从列表中移除该任务项
    taskList.removeChild(li);

    // 更新空状态显示
    updateEmptyState();
}
```

**实现要点**：
- 在删除任务后调用 `updateEmptyState()`
- 删除最后一个任务后，空状态自动显示

**（5）页面加载时初始化空状态**
```javascript
// 页面加载完成后初始化空状态
updateEmptyState();
```

**实现要点**：
- 在脚本末尾调用 `updateEmptyState()`
- 页面初始加载时，列表为空，空状态显示

### 实现细节

#### 技术决策说明

**1. 为什么使用独立的空状态元素？**
- ✅ **结构清晰**：空状态提示是独立元素，不混入任务列表
- ✅ **易于控制**：可以独立控制显示/隐藏
- ✅ **样式区分**：可以通过 CSS 与列表项明显区分
- ✅ **易于扩展**：未来可以在空状态中添加图标、按钮等

**2. 为什么使用 taskList.children.length 判断？**
- ✅ **准确可靠**：直接获取子元素数量，判断是否有任务
- ✅ **性能良好**：直接访问 DOM 属性，无需遍历
- ✅ **语义清晰**：`children` 只包含元素节点，不包含文本节点
- ✅ **实时更新**：每次调用都获取最新状态

**3. 为什么在添加和删除后都调用 updateEmptyState()？**
- ✅ **双向同步**：添加和删除都会影响任务数量
- ✅ **自动响应**：用户操作后自动更新空状态
- ✅ **统一逻辑**：同一函数处理两种情况，代码简洁

**4. 为什么使用 display 而非 visibility？**
- ✅ **完全隐藏**：`display: none` 不占用页面空间
- ✅ **用户体验**：空状态隐藏后，列表项可以占据更多空间
- ✅ **布局稳定**：使用 `display` 不会影响页面布局计算

**5. 空状态样式设计考虑**
- **虚线边框**：与列表项实线边框区分，表示"空"的状态
- **居中显示**：提示文字居中，视觉平衡
- **充足留白**：`padding: 60px 20px` 使提示更突出
- **柔和颜色**：浅灰色文字和背景，不抢眼
- **负边距调整**：`margin-top: -11px` 与任务列表无缝衔接

#### 核心技术要点
1. **独立空状态元素**：使用 `<div>` 创建空状态提示区域
2. **条件显示逻辑**：根据任务数量控制显示/隐藏
3. **状态同步机制**：在添加和删除操作后更新空状态
4. **视觉区分设计**：使用虚线边框和居中布局
5. **初始化处理**：页面加载时显示空状态
