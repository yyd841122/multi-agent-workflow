# Todo List 任务拆分清单

## 任务优先级说明

- **P0 (最高优先级)**: 核心基础功能，阻塞其他任务
- **P1 (高优先级)**: 主要功能，应该优先完成
- **P2 (中优先级)**: 重要功能，可并行开发
- **P3 (低优先级)**: 优化和完善，可最后完成

### T001: 创建项目基础文件结构 [P0]

**状态**: completed

**任务描述**:
创建 HTML、CSS、JavaScript 三个基础文件，建立文件间的引用关系。

**验收标准**:
1. 项目根目录存在 `index.html` 文件
2. 项目根目录存在 `style.css` 文件
3. 项目根目录存在 `script.js` 文件
4. HTML 文件正确引用 CSS 文件
5. HTML 文件正确引用 JavaScript 文件
6. 在浏览器打开 HTML 文件不报错

**依赖关系**:
- 无依赖

**预估工时**: 15 分钟

**技术要点**:
- 使用 HTML5 boilerplate
- CSS 通过 `<link>` 引入
- JS 通过 `<script>` 引入，放在 `</body>` 前

### T004: 实现添加任务功能 [P1]

**状态**: completed
实现点击添加按钮，将输入框内容添加到任务列表的功能。

**验收标准**:
1. 点击添加按钮时，输入框内容被添加到任务列表
2. 新任务出现在列表底部
3. 添加后输入框自动清空
4. 添加后输入框自动获得焦点，方便继续添加
5. 每个任务项包含：任务文本和删除按钮

**依赖关系**:
- 依赖: T002
- 被依赖: T005, T006

**预估工时**: 30 分钟

**技术要点**:
- 使用 `document.getElementById` 获取元素
- 使用 `document.createElement` 创建列表项
- 使用 `appendChild` 添加到列表
- 使用 `value` 属性获取和清空输入框
- 使用 `focus()` 方法聚焦输入框

### T005: 实现空输入校验 [P1]

**状态**: completed
添加输入校验，不允许添加空任务或仅包含空格的任务。

**验收标准**:
1. 输入框为空时点击添加，不添加任务
2. 仅输入空格时点击添加，不添加任务
3. 空输入时有用户提示（如：输入框抖动或显示提示文字）
4. 提示方式友好，不使用原生 alert
5. 校验通过后正常添加任务

**依赖关系**:
- 依赖: T004, T003
- 被依赖: T008

**预估工时**: 25 分钟

**技术要点**:
- 使用 `trim()` 方法去除首尾空格
- 检查字符串长度
- 添加临时提示样式或文字
- 使用 `setTimeout` 移除提示效果

### T006: 实现删除任务功能 [P1]

**状态**: completed
为每个任务添加删除按钮，点击后从列表中移除该任务。

**验收标准**:
1. 每个任务右侧有删除按钮（使用 × 或删除图标）
2. 点击删除按钮，该任务从列表中移除
3. 删除操作有确认提示或动画效果（可选）
4. 删除按钮样式明显，易于点击
5. 删除后列表重新渲染，无残留

**依赖关系**:
- 依赖: T004
- 被依赖: T008

**预估工时**: 25 分钟

**技术要点**:
- 为删除按钮绑定点击事件
- 使用 `remove()` 或 `removeChild()` 删除元素
- 使用事件委托（推荐）或为每个按钮单独绑定
- 删除按钮使用 HTML 实体 `&times;` 或文字

### T007: 实现任务完成状态切换 [P1]

**状态**: completed
点击任务文本可切换任务的完成/未完成状态。

**验收标准**:
1. 点击任务文本区域（非删除按钮），切换完成状态
2. 完成的任务有明显视觉区别（删除线、变灰等）
3. 可以在完成和未完成状态间反复切换
4. 切换状态不影响删除功能
5. 点击删除按钮不会触发完成状态切换

**依赖关系**:
- 依赖: T003, T004
- 被依赖: T008

**预估工时**: 30 分钟

**技术要点**:
- 为任务文本添加点击事件
- 使用 CSS class 标记完成状态（如 `.completed`）
- 使用 `classList.toggle()` 切换 class
- 使用 CSS `text-decoration: line-through` 添加删除线
- 使用 CSS `color: #999` 降低已完成任务的视觉权重
